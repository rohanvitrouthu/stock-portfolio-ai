## Starting Point

The project already had:

- Python package under `src/stock_portfolio_ai/`
- Market Data Agent using `yfinance`
- Fundamental Analyst Agent using `yfinance`
- Rust Axum gateway for index search and stock rows
- Wikipedia-based index constituent scraping
- quote enrichment through Yahoo Finance and a Python `yfinance` fallback
- initial docs and roadmap

The main open question was how to move from raw data and individual analyst tools toward investor-facing value.

## Decision: Defer Quote Latency Optimization

We reviewed quote enrichment latency for market cap and price. The gateway already renders constituents first and enriches quote data later, but quote enrichment can still take 30 seconds or more when external providers are slow.

The decision was to defer deeper latency work until after the MVP is more complete.

Reasoning:

- the current UI remains usable because constituents render before quote enrichment
- premature optimization could distract from the investor workflow
- full-service bottlenecks should be measured once the research and sector workflow exists
- future fixes likely involve caching, in-flight request coalescing, provider upgrades, and persistence

## Shared Analyst Report Schema

### What Was Built

Added:

- `src/stock_portfolio_ai/reports.py`

Core models:

- `AnalystReport`
- `EvidenceItem`
- `AgentType`
- `Rating`

### Why It Matters

Without a shared schema, each agent would return a different free-form structure. That would make portfolio management and supervisor orchestration fragile because downstream code would need to parse text or guess field meanings.

The schema gives every analyst a common contract:

- symbol
- agent type
- rating
- confidence
- summary
- key points
- risks
- evidence

### Integration

The Fundamental Analyst Agent now exposes:

- `analyze_symbol_report(symbol)`

This returns a validated `AnalystReport` with fundamental evidence from:

- valuation metrics
- profitability metrics
- financial statements
- cash flow data

## Discussion: Sectoral Analysis and Investor Value

The prior NIFTY-50 sectoral-analysis notebooks were reviewed at a high level. They include:

- EDA for Automobile sector stocks
- Random Forest regression work
- LSTM analysis

The conclusion was that this work is valuable, but raw closing-price prediction should not be the primary investor-facing feature.

Better investor-facing questions:

- Which sectors are improving or deteriorating?
- Which stocks are driving sector performance?
- Is sector performance broad-based or concentrated?
- Are fundamentals and technicals aligned?
- What are the risk, drawdown, volatility, and correlation characteristics?
- What should an investor buy, avoid, watch, or rebalance?

The ML notebooks should later become an evidence source, not the final recommendation engine.

## External Research Direction

We compared the product direction against public descriptions of institutional investment workflows:

- BlackRock Aladdin emphasizes portfolio-wide risk, exposure, stress testing, performance, and scenario analysis.
- Morgan Stanley materials emphasize asset allocation, volatility or VaR risk control, factor exposure, constraints, and portfolio construction.
- Jane Street describes a mix of quantitative analysis, market mechanics, machine learning, domain expertise, and rigorous research.

Takeaway:

The product should combine data, signals, risk, and portfolio reasoning. It should not rely on a single price-prediction model.

## Common Data Model and Sector Reference Data

### Decision

Sector belongs to the stock reference model, not to each index membership row.

Reasoning:

- a stock can belong to multiple indices
- a company normally has one canonical sector in our taxonomy
- sector classification changes far less frequently than prices
- index membership and quote data should be modeled separately

## Current Important Limitations

- No persistent database has been introduced yet.
- Sector coverage is currently local and incomplete for all supported indices.
- Quote enrichment latency remains dependent on Yahoo/yfinance.
- Some tests depend on live market data and may skip or fail if providers are unavailable.
- Agent ratings are heuristic and not backtested.
- The prior sector-analysis notebooks are present but not yet productized.

## Recommended Next Feature

Build a Sector Analyst layer.

Minimal useful scope:

- group index constituents by sector
- compute sector-level metrics from existing technical indicators
- produce a sector snapshot per index
- emit a sector-level `AnalystReport` or a sibling schema

Suggested sector metrics:

- stock count
- average 1-month return
- average 3-month return
- average RSI
- percentage of stocks above 50-day moving average
- volatility
- top contributors
- weakest stocks

This is the best next bridge between:

- sectoral-analysis notebooks
- current gateway index data
- fundamental and technical analyst reports
- future portfolio allocation
